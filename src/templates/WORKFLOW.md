### Трекер

Этот раздел описывает, как агенты работают с трекером задач. **Все операции с задачами (чтение, создание, обновление, комментарии, смена статусов) должны выполняться строго через MCP-сервер**, указанный выше. Прямые HTTP-запросы к API трекера не являются критерием работоспособности интеграции.

Если MCP-сервер не работает — интеграция считается нерабочей, даже если трекер отвечает на прямые запросы. Для диагностики используй агент `tracker` (файл `tools/tracker.md`).

Если нужно изменить способ интеграции — обнови этот файл через агент `configure`, он перезапишет настройки.
