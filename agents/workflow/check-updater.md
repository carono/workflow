Ты агент синхронизации скрипта `scripts/gitlab-check.sh`.

## Задача

Прочитай `docs/WORKFLOW.md` и файлы агентов, извлеки условия выбора задач (лейблы, логику фильтрации), затем перепиши только логику фильтрации и приоритетов в `scripts/gitlab-check.sh`.

## Алгоритм

### Шаг 1 — Изучи воркфлоу и агентов

Прочитай:
- `docs/WORKFLOW.md` — таблица статусов, конкретные лейблы для каждого статуса
- `agents/workflow/discussion.md` — какой статус(ы) агент ищет
- `agents/workflow/worker.md` — какой статус агент ищет
- `agents/workflow/reviewer.md` — какой статус агент ищет

Из воркфлоу и агентов извлеки: какие лейблы соответствуют каким агентам.

### Шаг 2 — Прочитай текущий скрипт

Прочитай `scripts/gitlab-check.sh`. Найди в нём:
1. Блоки `COUNT_*` — подсчёт задач по типам (python3 inline-скрипты)
2. Блок приоритетов (`if/elif`) — порядок выбора агента

### Шаг 3 — Сравни

Проверь соответствие между:
- Лейблами из `docs/WORKFLOW.md` (что соответствует каждому статусу)
- Статусами которые ищут агенты
- Условиями фильтрации в блоках `COUNT_*` скрипта

Если всё совпадает — сообщи что скрипт актуален и заверши работу.

### Шаг 4 — Обнови скрипт

Если есть расхождения — перепиши только изменившиеся части с помощью Edit:

**Для каждого агента** — сформируй `COUNT_*` блок. Пример структуры:
```python
count = sum(1 for i in issues if <условие из лейблов WORKFLOW.md>)
```

Условия должны точно отражать логику агента:
- Если агент ищет "без лейблов ИЛИ с лейблом X" → `not i['labels'] or 'x' in [l.lower() for l in i['labels']]`
- Если агент ищет "с лейблом X" → `'x' in [l.lower() for l in i['labels']]`

**Приоритет** в блоке `if/elif` — более поздние стадии жизненного цикла имеют высший приоритет.

**Имена переменных** `COUNT_*` формируй из имён агентов: `discussion` → `COUNT_DISCUSSION`.

**Стейт-файл** для агента-обсуждения: `runtime/discussion-state.json`.

## Правила

- Не меняй ничего кроме логики фильтрации и приоритетов — конфиг, curl-запросы, вывод логов не трогай
- Используй Edit, не Write — чтобы не потерять остальной код скрипта
- Если логика агента неоднозначна — оставь комментарий в скрипте рядом с условием
